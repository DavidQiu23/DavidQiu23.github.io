## 實作概念
為了實作網頁上的即時進度條，我們可以使用兩個通訊通道與後端溝通：
1. **動作通道 (Action Channel)**：負責發送實際動作請求的 Request（例如：複製檔案、匯入資料）。
2. **進度通道 (Progress Channel)**：採用 **長輪詢 (Long Polling)** 機制，負責獲取當前的進度資訊。

### 運作流程
1. 前端發送動作請求。
2. 後端接收請求後，建立一個 `Progress` 物件並存入全域字典中，同時指定該任務的總量（Total）。
3. 前端同步啟動長輪詢，持續向後端請求進度資料，並更新進度條 UI。
4. 當進度達到 100% 或動作請求完成（Response）後，結束長輪詢並釋放後端的 `Progress` 物件。

## 後端實作 (.NET C#)

### 進度管理工具類別 (ProgressUtil)

```csharp
public class ProgressUtil
{
    public class Progress
    {
        public Progress(int total) {
            this.total = total;
        }
        /// <summary>任務總量 (0 表示查無實體)</summary>
        public int total { get; set; }
        /// <summary>目前已完成數量</summary>
        public int now { get; set; } = 0;
    }

    // 全域字典：Key 為 UserSeq (使用者識別碼)，Value 為 Progress 進度物件
    private static readonly Dictionary<string, Progress> progressDict = new Dictionary<string, Progress>(); 

    /// <summary>建立新的進度物件</summary>
    public static void CreateProgress(int total)
    {
        var userSeq = SessionUtil.Current.UserSeq;
        if (!progressDict.ContainsKey(userSeq))
            progressDict.Add(userSeq, new Progress(total));
    }

    /// <summary>推進單步進度</summary>
    public static void AddProgress()
    {
        var userSeq = SessionUtil.Current.UserSeq;
        if (progressDict.ContainsKey(userSeq))
            progressDict[userSeq].now += 1;
    }

    /// <summary>動態增加任務總量 (適用於多階段任務)</summary>
    public static void AddTotal(int count)
    {
        var userSeq = SessionUtil.Current.UserSeq;
        if (progressDict.ContainsKey(userSeq))
            progressDict[userSeq].total += count;
    }

    /// <summary>獲取當前進度</summary>
    public static Progress GetProgress()
    {
        if (progressDict.TryGetValue(SessionUtil.Current.UserSeq, out Progress progress))
            return progress;
        return new Progress(0);
    }

    /// <summary>任務完成，移除進度物件</summary>
    public static void RemoveProgress()
    {
        progressDict.Remove(SessionUtil.Current.UserSeq);
    }

    /// <summary>異常處理：直接將進度設為 100% 以終止 UI 等待</summary>
    public static void ErrorProgress()
    {
        var userSeq = SessionUtil.Current.UserSeq;
        if (progressDict.ContainsKey(userSeq))
            progressDict[userSeq].now = progressDict[userSeq].total;
    }
}
```

## 前端實作 (JavaScript / Axios)

前端需同時觸發動作 API 與進度查詢函數。

```javascript
// 1. 發送動作請求 (例如複製任務)
axios.post('@Url.Action("Copy")', 
    { from: this.copy.from, to: this.copy.to }, 
    { headers: this.headers }
).then((res) => {
    this.isResponse = true; // 標記為已回應
    if (res.data > 0) {
        this.$alert('操作成功', '提示', {
            callback: () => {
                this.dialogCopyVisible = false;
                this.LoadTableData();
            }
        });
    } else {
        this.$alert('操作失敗', '錯誤');
    }
});

// 2. 啟動進度查詢
this.GetProgress();

// --- 查詢函數實作 ---
function GetProgress() {
    axios.get('@Url.Action("GetProgress", "Home")')
    .then((res) => {
        // 若任務尚未結束且後端有進度資料
        if (res.data.total !== 0 && !this.isResponse) {
            // 計算並更新百分比
            this.progressPercentage = Math.round((res.data.now / res.data.total) * 100);
            
            // 若未達 100%，則繼續下一次輪詢
            if (res.data.now < res.data.total) {
                setTimeout(() => this.GetProgress(), 500); // 延遲半秒後繼續請求
            }
        } else if (res.data.total === 0 && !this.isResponse) {
            // 後端物件尚未建立，持續等待
            setTimeout(() => this.GetProgress(), 500);
        }
    });
}
```
